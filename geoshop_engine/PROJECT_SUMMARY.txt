PROJECT OVERVIEW
───────────────────────────────────────────────────────────────────────────────
A production-ready, real-time geolocation-based shop intelligence platform
that aggregates data from multiple sources with confidence-based scoring and
MongoDB Atlas storage.

                        🚀 QUICK START GUIDE
═══════════════════════════════════════════════════════════════════════════════

1. INSTALL DEPENDENCIES
   $ pip install -r requirements.txt

2. CONFIGURE ENVIRONMENT
   Edit .env with MongoDB Atlas credentials (pre-configured for defaults)

3. TEST PIPELINE (OFFLINE)
   $ python main.py demo
   Expected: Processes 5 mock records → 3 unique shops (80%+ confidence)

4. START API SERVER
   $ python run_api.py
   Expected: Server runs on http://localhost:8000

5. OPEN FRONTEND
   Option A: http://localhost:8000
   Option B: Open frontend/index.html directly
   Expected: Interactive dashboard with live statistics

6. TRIGGER UPDATES
   • Dashboard: Click "Real-time Update" or "Trigger Sync"
   • API: curl -X POST http://localhost:8000/api/sync/trigger
   • CLI: python main.py run

                       🏗️ ARCHITECTURE HIGHLIGHTS
═══════════════════════════════════════════════════════════════════════════════

LAYERED DESIGN
  • Presentation: HTML/JS Dashboard with REST API calls
  • API Layer: FastAPI with FastAPI with 8 endpoints
  • Business Logic: Processing pipeline (normalize, match, score)
  • Data Layer: MongoDB with indexed collections
  • Integration: Multiple data sources with fallback

CONFIDENCE SCORING
  • Multi-factor algorithm combining:
    - Data completeness (address, phone, website, hours)
    - Source reliability (OSM > DataGov > OneMap)
    - Match quality (string similarity + proximity)
    - Coordinate validation (within Singapore bounds)
  • Result: 0-100% confidence score
  • Application: Filters real-time database updates at ≥50%

REAL-TIME CAPABILITY
  • Frontend UI triggers immediate update
  • Pipeline fetches from all 3 sources
  • Processes within 30 seconds
  • Compares with existing database
  • Updates only if confidence ≥ threshold
  • User sees results in dashboard

OFFLINE RESILIENCE
  • Works without internet (mock data)
  • Graceful API failure handling
  • Database connection optional
  • Test any component independently
